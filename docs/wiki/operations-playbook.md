# 運營手冊 —「AI物聯網實驗室」平台管理員(admin)

這份手冊寫給平台的經營者(你)。角色分工:**admin(你)**負責發帳號、
控成本、調設定;**教師(客戶)**只管建班、發班級代碼、看報表;
**學生**只管上課。老師和學生永遠不需要碰任何技術設定。

## 一、初始部署(一次性)

伺服器環境變數(只有這些需要在部署時設,其餘全部在管理後台 UI 調):

| 變數 | 用途 |
|---|---|
| `VELXIO_ADMIN_EMAIL` / `VELXIO_ADMIN_PASSWORD` | 你的 admin 帳號(重啟即生效;忘記密碼就改 env 重啟) |
| `VELXIO_OPENAI_BASE_URL` / `VELXIO_OPENAI_API_KEY` | AI 上游(你的中轉/金鑰,**成本來源**) |
| `VELXIO_SECRET_KEY` | 登入 token 簽章金鑰(不設會自動產生存在 data 目錄) |
| `VELXIO_DATA_DIR` | SQLite 資料目錄(Docker 內 `/app/data`,**務必掛 volume 並定期備份**) |

部署後第一件事:登入 `/admin` → 平台設定 → 依你的商業模式核對每一項
(見下節)。模型/額度等的 env 值只是「出廠預設」,UI 儲存後以 UI 為準。

## 二、平台設定(管理後台 → 平台設定)

| 設定 | 建議值(經營模式:帳號由你發放) |
|---|---|
| AI 模型 | `gpt-5.6-luna`(預設)。換模型前先用自己帳號在編輯器試一輪 |
| 推理強度 | `high`(最聰明;要省成本可降 `medium`) |
| 學生每週額度 | 預設 2,000,000 tokens。估算:一次完整的 AI 搭建+除錯對話約 5–15 萬 tokens,2M ≈ 每週 15–40 次紮實使用 |
| 教師每週額度 | 預設 4,000,000(老師備課用量較大) |
| 開放自助註冊 | **建議關閉** — 帳號全部由你批量發放,平台就是純封閉商用系統;若想開放試用再打開 |
| 允許自選模型/強度 | **建議關閉** — 全平台統一,成本可控 |
| 允許自帶 API Key | 看你商業模式:關閉 = 所有用量都走你的金鑰(好記帳);開啟 = 進階客戶可自帶(不計你的量) |

每項儲存即全平台生效,不用重啟。

## 三、新客戶開通 SOP(約 3 分鐘)

以「某校老師帶 2 個班共 60 名學生」為例:

1. **開教師帳號**:管理後台 → 批量建立 → 身分「教師」、數量 1、
   前綴如 `wang`、網域用客戶好記的(如 `xx-school.tw`)→ 下載 CSV。
2. **請老師建班**:把教師帳密交給老師,老師登入 → 教學管理 →
   建立班級 → 得到 6 碼班級代碼(每班一碼)。
3. **開學生帳號**:批量建立 → 身分「學生」、數量 30、前綴 `s7a`
   (七年甲班)、**填入班級代碼**(自動入班)、必要時設定該批額度
   → 下載 CSV 交給老師(印出來發或貼在班群)。第二個班用前綴 `s7b`
   再跑一次。
4. 完成。學生登入即看到課程,老師報表即時有全班名單。

命名建議:前綴用「學校縮寫+年級班別」(`s7a`、`s7b`),網域統一
(`school.local` 或客戶校名),之後在帳號管理搜尋前綴就能整批找到。

## 四、每週例行(5 分鐘)

- 開管理後台看總覽:**本週 AI tokens** 是你的即時成本錶
  (成本 ≈ tokens × 你上游的單價)。
- 帳號表按用量排查:進度條變紅(≥95%)的學生 = 重度使用者;
  視情況調高該生額度,或提醒老師。
- 額度每週一(UTC)自動歸零重算,不用手動處理。

## 五、常見狀況處置

| 狀況 | 處置 |
|---|---|
| 學生/老師忘記密碼 | 帳號管理 → 該帳號「重設密碼」→ 新密碼只顯示一次,抄給對方 |
| 學生 AI 用完了 | 學生會看到 429 中文提示。你可:調高其額度 / 請他等週一 / 不處理(課程與模擬器**不受影響**,只有 AI 助教暫停) |
| 客戶合約到期 | 帳號管理逐一刪除(搜前綴整批處理),或先把該批額度設為 0 保留資料 |
| 有人濫用 AI | 把該帳號額度設 0(立即斷供,不刪資料) |
| 想換更便宜的模型 | 平台設定改模型 → 儲存 → 自己開編輯器問 AI 一題確認品質 |
| 疑似被暴力試密碼 | 內建 10 次/分/IP 限流已擋;正式站建議 nginx 再加一層 |

## 六、成本與安全備忘

- **成本上限公式**:最壞情況週成本 ≈(學生數×學生額度 + 教師數×教師額度)× 上游單價。發帳號前先算一次。
- **備份**:`$VELXIO_DATA_DIR`(內含 `velxio-cloud.db` 與 secret_key)
  就是全部狀態,cron 每日 copy 一份即可。
- admin 密碼夠長;`/admin` 沒有第二道防線,誰拿到 admin 帳密就是平台主人。
- 批量建立回傳的密碼**只顯示一次**(資料庫只存雜湊)— 當場下載 CSV。
